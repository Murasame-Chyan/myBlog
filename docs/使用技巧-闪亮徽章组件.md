# 使用技巧：闪亮徽章组件

## 概述

闪亮徽章是一个纯前端组件，包含卡片缩略图 + 全息预览模态框。效果移植自宝可梦闪卡 CSS Demo，通过弹簧物理模拟 3D 倾斜、多层彩虹光泽和镜面高光。用于给用户颁发纪念徽章、成就徽章等，点击展开全息卡片预览。

**三文件组件，零依赖框架，接入任意页面即可使用。**

---

## 素材目录约定

素材分为两部分，通过 **`badgeId` 命名约定** 自动关联：

```
src/main/resources/static/images/badge/
  icon/                         ← 缩略图（36px 按钮用）
    re-zero-icon.jpg            ← xxx-icon.jpg
    your-badge-icon.png
  fullbadge/                    ← 全息大卡（模态框预览用）
    re-zero-full-1.jpg          ← xxx-full-1.jpg（序列第一张）
    re-zero-full-2.jpg          ← xxx-full-2.jpg（如果有多个版本）
```

**命名规则：**

| 类型 | 路径模式 | 说明 |
|------|----------|------|
| 缩略图 | `/images/badge/icon/{badgeId}-icon.jpg` | 显示在按钮中，宽 36px |
| 全息卡 | `/images/badge/fullbadge/{badgeId}-full-{n}.jpg` | n 从 1 开始，对应徽章序列 |

同一个 `badgeId` 通过前缀将缩略图和全息大卡关联起来，JS 自动拼路径，无需手写每张图的 URL。

---

## 文件清单

| 文件 | 路径 | 职责 |
|------|------|------|
| 样式 | `src/main/resources/static/css/badge-card.css` | 全息卡光泽层、缩略图扫光动画、模态框遮罩 |
| 脚本 | `src/main/resources/static/js/badge-card.js` | 弹簧物理指针交互、模态框开关、徽章注册表 + 路径拼装 |
| 缩略图素材 | `src/main/resources/static/images/badge/icon/` | 徽章缩略图，建议 ~100×140 小图 |
| 全息卡素材 | `src/main/resources/static/images/badge/fullbadge/` | 徽章大图，建议 500×700（Pokemon 卡比例 0.718） |

## 接入步骤

### 1. 准备素材

把缩略图和全息大卡分别放入 `badge/icon/` 和 `badge/fullbadge/`，按约定命名：

```bash
# 示例：添加一个徽章，badgeId = "re-zero"
cp thumb.jpg  src/main/resources/static/images/badge/icon/re-zero-icon.jpg
cp full1.jpg  src/main/resources/static/images/badge/fullbadge/re-zero-full-1.jpg
cp full2.jpg  src/main/resources/static/images/badge/fullbadge/re-zero-full-2.jpg  # 可选
```

### 2. 注册徽章元数据

在 `badge-card.js` 顶部的 `BADGES` 对象中声明徽章名称和全息卡序列数量：

```javascript
const BADGES = {
    "re-zero": {
        name: "Re:0 第四季 纪念徽章",
        count: 1            // fullbadge 序列数量（有多少张 xxx-full-N）
    },
    "your-badge": {
        name: "你的徽章",
        count: 3            // 有 3 张全息卡变体
    }
};
```

**JS 会自动拼出路径：**
- 缩略图：`/images/badge/icon/re-zero-icon.jpg`
- 全息卡 #1：`/images/badge/fullbadge/re-zero-full-1.jpg`
- 全息卡 #2：`/images/badge/fullbadge/your-badge-full-2.jpg`（依此类推）

### 3. 在页面 `<head>` 引入 CSS

```html
<link rel="stylesheet" href="/css/badge-card.css">
```

### 4. 在目标位置放置缩略图按钮

```html
<button type="button" class="profile-badge-thumb"
        data-badge-id="re-zero"
        aria-label="查看 Re:0 第四季 纪念徽章">
    <img src="/images/badge/icon/re-zero-icon.jpg" alt="Re:0 第四季 纪念徽章">
</button>
```

**关键属性：**
- `data-badge-id` — 对应 JS 注册表中的 key，也用于拼装全息卡路径
- `aria-label` — 无障碍标签
- `<img>` 的 `src` — 按约定指向 `badge/icon/{badgeId}-icon.jpg`

### 5. 在 `</body>` 前插入模态框 DOM 和 JS

```html
<!-- 闪亮徽章预览 Modal -->
<div id="badgeModal" class="badge-modal" hidden aria-hidden="true" role="dialog" aria-label="徽章预览">
    <div class="badge-modal__backdrop"></div>
    <div class="badge-modal__content">
        <div class="card">
            <div class="card__rotator">
                <div class="card__front">
                    <img id="badgeCardImg" src="" alt="">
                    <div class="card__shine"></div>
                    <div class="card__glare"></div>
                </div>
            </div>
        </div>
        <div class="badge-modal__name" id="badgeModalName"></div>
    </div>
    <button type="button" class="badge-modal__close" aria-label="关闭">&times;</button>
</div>
<script src="/js/badge-card.js"></script>
```

**DOM 契约（不要改 `id`）：**

| id | 说明 |
|----|------|
| `#badgeModal` | JS 通过它查找模态框，`hidden` 属性控制显隐 |
| `#badgeCardImg` | JS 写入全息卡的 `.src` 和 `.alt` |
| `#badgeModalName` | JS 写入徽章名称文字 |

## 动态替换注册表（接后端 API）

JS 把 `BADGES` 挂载到了 `window.BADGES`，外部可以随时覆盖：

```javascript
// 从 API 获取徽章数据后动态注入
fetch("/api/user/badges")
    .then(res => res.json())
    .then(data => {
        window.BADGES = data;  // 完全替换硬编码注册表
    });
```

**后端返回的 JSON 格式需与注册表结构一致：**

```json
{
    "re-zero": { "name": "Re:0 第四季 纪念徽章", "count": 1 },
    "anniversary": { "name": "一周年纪念徽章", "count": 2 }
}
```

## 多徽章排版

同一行放多个徽章，重复 `<button>` 并指定不同 `data-badge-id`：

```html
<div style="display:flex; gap:6px; align-items:center;">
    <button type="button" class="profile-badge-thumb" data-badge-id="re-zero" aria-label="徽章A">
        <img src="/images/badge/icon/re-zero-icon.jpg" alt="徽章A">
    </button>
    <button type="button" class="profile-badge-thumb" data-badge-id="anniversary" aria-label="徽章B">
        <img src="/images/badge/icon/anniversary-icon.jpg" alt="徽章B">
    </button>
</div>
```

## 完整的 Thymeleaf 接入示例

```html
<!-- 假设 profileUser.badges 是 List<BadgeVO>，BadgeVO 含 id/name/count 字段 -->
<th:block th:each="badge : ${profileUser.badges}">
    <button type="button" class="profile-badge-thumb"
            th:data-badge-id="${badge.id}"
            th:attr="aria-label='查看 ' + ${badge.name}">
        <!-- 缩略图按约定路径：badge/icon/{id}-icon.jpg -->
        <img th:src="@{/images/badge/icon/__${badge.id}__-icon.jpg}"
             th:alt="${badge.name}">
    </button>
</th:block>
```

底部注入后端数据覆盖注册表：

```html
<script th:inline="javascript">
    window.BADGES = /*[[${badgesMap}]]*/ {};
</script>
<script src="/js/badge-card.js"></script>
```

## 交互规格

| 操作 | 行为 |
|------|------|
| 鼠标划过缩略图 | 扫光持续循环，hover 上浮 + 粉紫辉光 |
| 点击缩略图 | 模态框淡入，背景模糊遮罩，全息卡居中 |
| 鼠标在卡片上移动 | 3D 倾斜（~14° 最大）+ 多层光泽跟随 + 镜面高光 |
| 鼠标离开卡片 | 500ms 后弹簧缓动回正 |
| 点击 × / 点击背景遮罩 / 按 Esc | 模态框关闭，焦点回到缩略图 |
| 模态框打开时 | body 滚动锁定（`overflow: hidden`） |

## CSS 自定义属性速查

全息卡的视觉状态全部通过 CSS 变量驱动，JS 只负责写入这些变量：

| 变量 | 说明 | 范围 |
|------|------|------|
| `--tilt-left-right` | Y 轴旋转(deg) | ~ -14~14 |
| `--tilt-up-down` | X 轴旋转(deg) | ~ -14~14 |
| `--pointer-x` | 指针 X 位置(%) | 0~100 |
| `--pointer-y` | 指针 Y 位置(%) | 0~100 |
| `--pointer-from-center` | 距中心距离(0~1) | 0~1 |
| `--effect-intensity` | 整体效果强度(0~1) | 0~1 |
| `--background-x` | 光泽层水平偏移(%) | 37~63 |
| `--background-y` | 光泽层垂直偏移(%) | 33~67 |
| `--foil` | 金属箔纹理图片 URL | 任意 url() |
| `--space` | 彩虹条纹间距(%) | 建议 6% |
| `--angle` | 暗纹角度(deg) | 建议 133deg |

## JS 内部 API

### `getIconPath(badgeId)` → `string`
返回缩略图路径。等价于 `/images/badge/icon/{badgeId}-icon.jpg`。

### `getFullPath(badgeId, n)` → `string`
返回第 n 张全息卡路径。等价于 `/images/badge/fullbadge/{badgeId}-full-{n}.jpg`。

### `window.BADGES`
全局可写注册表。格式：`{ [badgeId]: { name: string, count: number } }`。

## 注意事项

1. **图片比例** — 卡片容器宽高比固定为 0.718（Pokemon TCG 卡标准），大图用 `object-fit: cover` 填充
2. **缩略图尺寸** — 36px 宽是设计值，改大需同步调 CSS 中 `.profile-badge-thumb` 的 `width`
3. **icon / fullbadge 必须对应** — 缩略图和全息卡通过 `badgeId` 前缀关联，文件名要严格遵守 `{badgeId}-icon` / `{badgeId}-full-{n}` 格式
4. **foil 纹理** — `--foil` 默认引用外部 CDN（Imgur），生产环境建议改为本地托管的纹理图
5. **性能** — 动画循环在弹簧到达目标后自动停止，不影响页面性能
6. **单实例** — 目前设计只支持页面中**一个** `#badgeModal`，多页面各自插入即可
