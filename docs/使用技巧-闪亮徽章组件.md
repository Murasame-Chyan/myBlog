# 使用技巧：闪亮徽章组件

## 概述

闪亮徽章是一个纯前端组件，包含卡片缩略图 + 全息预览模态框。效果移植自宝可梦闪卡 CSS Demo，通过弹簧物理模拟 3D 倾斜、多层彩虹光泽和镜面高光。用于给用户颁发纪念徽章、成就徽章等，点击展开全息卡片预览。

**三文件组件，零依赖框架，接入任意页面即可使用。**

---

## 文件清单

| 文件 | 路径 | 职责 |
|------|------|------|
| 样式 | `src/main/resources/static/css/badge-card.css` | 全息卡光泽层、缩略图扫光动画、模态框遮罩 |
| 脚本 | `src/main/resources/static/js/badge-card.js` | 弹簧物理指针交互、模态框开关、徽章注册表 |
| 素材 | `src/main/resources/static/images/badges/*.jpg` | 徽章图片，建议 500×700 左右（Pokemon 卡比例 0.718） |

---

## 接入步骤

### 1. 准备徽章图片

将徽章图片放入 `src/main/resources/static/images/badges/`，文件名使用 **ASCII 字符**，避免 URL 编码问题。

```
src/main/resources/static/images/badges/
  re-zero-s4.jpg        ← 示例
  your-badge.png
  another-badge.webp
```

### 2. 在页面 `<head>` 引入 CSS

```html
<link rel="stylesheet" href="/css/badge-card.css">
```

### 3. 在目标位置放置缩略图按钮

拇指图用 BEM 类 `.profile-badge-thumb`，宽 36px、Pokemon 卡比例，带有循环扫光动画暗示"可点击查看全息卡"。

```html
<button type="button" class="profile-badge-thumb"
        data-badge-id="your-badge-id"
        aria-label="查看 XXX 徽章">
    <img src="/images/badges/your-badge.jpg" alt="XXX 徽章">
</button>
```

**关键属性：**
- `data-badge-id` — 对应 JS 注册表中的 key，点击时查找对应图片和名称
- `aria-label` — 无障碍标签
- `<img>` 的 `src` 既是缩略图也是后备显示

### 4. 在 `</body>` 前插入模态框 DOM 和 JS

```html
<!-- 闪亮徽章预览 Modal -->
<div id="badgeModal" class="badge-modal" hidden aria-hidden="true" role="dialog" aria-label="徽章预览">
    <div class="badge-modal__backdrop"></div>
    <div class="badge-modal__content">
        <div class="card">
            <div class="card__rotator">
                <div class="card__front">
                    <img id="badgeCardImg" src="" alt="">
                    <div class="card__shine"></div>
                    <div class="card__glare"></div>
                </div>
            </div>
        </div>
        <div class="badge-modal__name" id="badgeModalName"></div>
    </div>
    <button type="button" class="badge-modal__close" aria-label="关闭">&times;</button>
</div>
<script src="/js/badge-card.js"></script>
```

**DOM 结构是固定契约，不要改 `id`：**

| id | 说明 |
|----|------|
| `#badgeModal` | JS 通过它查找模态框，`hidden` 属性控制显隐 |
| `#badgeCardImg` | JS 写入 `.src` 和 `.alt` |
| `#badgeModalName` | JS 写入徽章名称文字 |

## 注册徽章数据

所有徽章定义在 `badge-card.js` 顶部的 `BADGES` 对象中。接入后端接口时，替换这里即可。

```javascript
const BADGES = {
    "badge-id-1": {
        image: "/images/badges/your-badge.jpg",
        name: "徽章显示名称"
    },
    "badge-id-2": {
        image: "/images/badges/another.png",
        name: "另一个徽章"
    }
};
```

JS 会自动把 `BADGES` 挂载到 `window.BADGES`，方便外部动态替换：

```javascript
// 从 API 获取徽章数据后动态注入
fetch("/api/user/badges")
    .then(res => res.json())
    .then(data => {
        window.BADGES = data;  // 覆盖硬编码注册表
    });
```

## 多徽章排版

同一行放多个徽章只需重复 `<button class="profile-badge-thumb">`，每个指向不同 `data-badge-id`：

```html
<div style="display:flex; gap:6px; align-items:center;">
    <button type="button" class="profile-badge-thumb" data-badge-id="badge-1" aria-label="徽章A">
        <img src="/images/badges/a.jpg" alt="徽章A">
    </button>
    <button type="button" class="profile-badge-thumb" data-badge-id="badge-2" aria-label="徽章B">
        <img src="/images/badges/b.jpg" alt="徽章B">
    </button>
</div>
```

## 完整的 Thymeleaf 接入示例（接后端数据）

```html
<!-- 假设 profileUser.badges 是 List<BadgeVO>，BadgeVO 含 id/image/name 字段 -->
<th:block th:each="badge : ${profileUser.badges}">
    <button type="button" class="profile-badge-thumb"
            th:data-badge-id="${badge.id}"
            th:attr="aria-label='查看 ' + ${badge.name}">
        <img th:src="${badge.image}" th:alt="${badge.name}">
    </button>
</th:block>
```

同时在页面底部用内联 `<script>` 注入后端数据覆盖 `window.BADGES`：

```html
<script th:inline="javascript">
    // 覆盖硬编码 BADGES，改用服务端渲染的徽章数据
    window.BADGES = /*[[${badgesMap}]]*/ {};
</script>
<script src="/js/badge-card.js"></script>
```

## 交互规格

| 操作 | 行为 |
|------|------|
| 鼠标划过缩略图 | 扫光持续循环，hover 上浮 + 粉紫辉光 |
| 点击缩略图 | 模态框淡入，背景模糊遮罩，卡片居中 |
| 鼠标在卡片上移动 | 3D 倾斜（~14° 最大）+ 多层光泽跟随 + 镜面高光 |
| 鼠标离开卡片 | 500ms 后弹簧缓动回正 |
| 点击 × / 点击背景遮罩 / 按 Esc | 模态框关闭，焦点回到缩略图 |
| 模态框打开时 | body 滚动锁定（`overflow: hidden`） |

## CSS 自定义属性速查

全息卡的视觉状态全部通过 CSS 变量驱动，JS 只负责写入这些变量：

| 变量 | 说明 | 范围 |
|------|------|------|
| `--tilt-left-right` | Y 轴旋转(deg) | ~ -14~14 |
| `--tilt-up-down` | X 轴旋转(deg) | ~ -14~14 |
| `--pointer-x` | 指针 X 位置(%) | 0~100 |
| `--pointer-y` | 指针 Y 位置(%) | 0~100 |
| `--pointer-from-center` | 距中心距离(0~1) | 0~1 |
| `--effect-intensity` | 整体效果强度(0~1) | 0~1 |
| `--background-x` | 光泽层水平偏移(%) | 37~63 |
| `--background-y` | 光泽层垂直偏移(%) | 33~67 |
| `--foil` | 金属箔纹理图片 URL | 任意 url() |
| `--space` | 彩虹条纹间距(%) | 建议 6% |
| `--angle` | 暗纹角度(deg) | 建议 133deg |

## 注意事项

1. **图片比例** — 卡片容器宽高比固定为 0.718（Pokemon TCG 卡标准），图片用 `object-fit: cover` 填充，非标准比例会被裁切
2. **缩略图尺寸** — 36px 宽是设计值，改大需同步调 CSS 中 `.profile-badge-thumb` 的 `width`
3. **foil 纹理** — `--foil` 默认引用外部 CDN 图片（Imgur），生产环境建议改为本地托管的纹理图
4. **性能** — 动画循环在弹簧到达目标后自动停止（`requestAnimationFrame` 不闲置运行），不影响页面性能
5. **单实例** — 目前设计只支持页面中**一个** `#badgeModal`，多页面各自插入即可（不跨页面共享）
